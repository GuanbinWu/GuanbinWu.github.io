# Linux权限问题

1. 机器状态
```
passwd -S root 返回 root NP .. 或 root LK ...
sudo grep '^root:' /etc/shadow 返回root::或root:!:或root:*:
sudo visudo 中出现NOPASSWD类似字样
```

对于红帽系列的系统：
```
vim /etc/pam.d/su
```
返回
```auth           sufficient      pam_wheel.so trust use_uid
或
auth           required        pam_wheel.so use_uid
```
类似字段

2. 修复方法
对于红帽系，先将etc/pam.d/su高风险字段注释掉
然后在sudo visudo

1. 有时```sudo -i```能直接提权是因为不久前提权过，可以```sudo -k```清除缓存后再检查。
