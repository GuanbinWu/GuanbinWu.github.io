内网共享文件夹配置

  1. Apache2
systemctl status apache2
apache2ctl -S #查看监听站点

发现服务器自带Apache2服务，并监听了80端口。 2. 沿用现有的apache2服务，实现文件夹开放。

sudo a2enmod dav
sudo a2enmod dav_fs
sudo a2enmod auth_basic
sudo systemctl restart apache2
sudo apache2ctl configtest #此处检查配置文件语法，应当返回Syntax OK
  1. 配置文件夹
mkdir -p /data/srv/webdav
sudo chown -R www-data:www-data /data/srv/webdav
sudo echo hello > /data/srv/webdav/test.txt  #等会测试用
  1. 设置用户名和密码，此处采用Digest Auth，避免Basic Auth全程使用明文
sudo a2enmod auth_digest
sudo htdigest -c /etc/apache2/webdav.digest "WebDAV" username #此处完成用户名和密码创建
sudo vim /etc/apache2/conf-available/webdav.conf #修改配置文件

输入

Alias /webdav /data/srv/webdav
<Directory /data/srv/webdav>
    DAV On
    AuthType Digest  #这里与用户类型一致
    AuthName "WebDAV" 
    AuthUserFile /etc/apache2/webdav.digest #用户文件夹
    Require valid-user
    Options Indexes
    DirectoryIndex disabled
    LimitRequestBody 21474836480 #文件大小限制20G,0为无限制
    <LimitExcept GET OPTIONS PROPFIND>
        Require valid-user
    </LimitExcept>
</Directory>
ProxyTimeout 3600  #扩大Timeout，避免大文件传输时被kill
Timeout 3600

然后重启apache2

sudo apache2ctl configtest #先检查语法
sudo systemctl reload apache2
  1. 如果是从Basic用户迁移到了Digest用户，删一下遗留文件
sudo rm /etc/apache2/webdav.passwd
  1. 下载filebrowser实现文件托管。
curl -fsSL https://raw.githubusercontent.com/filebrowser/get/master/get.sh | bash #此命令需要与github畅通

如果连不上，可手动下载，路径为 https://github.com/filebrowser/filebrowser/releases/download/v2.63.14/

sudo tar -xzf /tmp/filebrowser-linux-amd64.tar.gz
sudo chmod +x filebrowser
filebrowser version #检查

移动到/usr/local/下，并设置好权限 7. 初始化filebrowser数据库

sudo mkdir -p /etc/filebrowser
cd /etc/filebrowser
filebrowser config init 
filebrowser -d /etc/filebrowser/filebrowser.db config set --root /srv/webdav #此命令在当前文件夹下初始化一个数据库.db

#创建管理员，后续浏览器登录也是用这个用户名和密码
filebrowser -d /etc/filebrowser/filebrowser.db users add admin M7f@9Qk!2Lp
  1. 将filebrowser做成系统服务
sudo vim /etc/systemd/system/filebrowser.service

内容:

[Unit]
Description=File Browser
After=network.target
[Service]
User=www-data      #这里和/data/src/webdav的拥有者一致
Group=www-data     #这里和/data/src/webdav的拥有者一致
ExecStart=/usr/local/filebrowser/filebrowser \
-d /etc/filebrowser/filebrowser.db \
-r /data/srv/webdav \
-a 127.0.0.1 \  # 只监听本机端口
--baseurl /files \ 
-p 8080
Restart=always
[Install]
WantedBy=multi-user.target

启用

sudo systemctl daemon-reload
sudo systemctl enable filebrowser
sudo systemctl start filebrowser
  1. Apache 反向代理 启用模块
sudo a2enmod proxy
sudo a2enmod proxy_http
sudo systemctl restart apache2
sudo vim /etc/apache2/sites-available/000-default.conf

<VirtualHost *:80>字段内加上

PreserveHost On
AllowEncodedSlashes NoDecode
ProxyPass        /files/ http://127.0.0.1:8080/files/ nocanon
ProxyPassReverse /files/ http://127.0.0.1:8080/files/
RedirectMatch 301 ^/files$ /files/

重载

sudo apache2ctl configtest
sudo systemctl reload apache2

  1. 测试 浏览器打开服务器ip/files，应当能够正常使用下载上传文件。

  2. 关于apache代理 如果不适用apache代理，在filebrowser服务里直接写-a 0.0.0.0监听全网即可。此时相当于暴露8080端口，外部浏览器也通过ip:8080可直接访问。显然此时/etc/apache2/sites-available/000-default.conf里的关于代理的配置都是无用功，注释掉完全没影响。

为了避免端口暴露，就需要将filebrowser服务改为只监听本机8080端口，通过apache2的80端口重定向到本机8080。 那么就需要改/etc/apache2/sites-available/000-default.conf 同时也需要改/etc/systemd/system/filebrowser.service。 此时sudo ss -lntp | grep :8080依然会看到是filebrowser在监听8080端口。但网页端只能通过80访问。

  1. 如何关闭端口
sudo systemctl stop filebrowser

即可，apache2依然运行，无副作用。
