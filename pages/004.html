Gromacs安装与使用

安装

参考教程： sob老师的博客 使用GROMACS 2024.6，并配置GPU加速。

  1. 准备cmake。最好直接用sudo apt install cmake或sudo dnf install cmake。但需要小心版本，GROMACS 2024.6需要3.18以上。如果需要手动安装新版cmake。过程如下： 进入https://cmake.org/download/。选择合适的版本，确定下载链接。
wget https://github.com/Kitware/CMake/releases/download/v4.0.1/cmake-4.0.1-linux-x86_64.tar.gz
tar -xvf xxx/cmake-xxx.tar.gz
cd xxx/cmake-xxx
./bootstrap
# Sob老师的教程里使用了 ./bootstrap --prefix=/sob/cmake3 -- -DCMAKE_USE_OPENSSL=OFF
# --prefix指定安装路径，-DCMAKE_USE_OPENSSL指定不依赖openssl。可自选
make -j #编译
sudo make install #安装

一切操作都可以在用户目录下完成，因为make install 用了sudo，所以最后的cmake会在/usr/local/下完成安装，可公共使用，自己编辑一下环境变量即可。

export PATH=/usr/local/cmake-4.0.1-linux-x86_64/bin:$PATH

  1. 安装fftw。可参考sob老师的博客。

  2. gmx安装：参考官方文档以及sob老师的博客，cmake一行记得把后面两个参数删掉，直接cmake即可。完毕后检查/usr/local/下是否有gromacs文件夹即可。

mkdir build
cd build
export CMAKE_PREFIX_PATH=/usr/local/fftw3.3.8
cmake3 .. -DCMAKE_INSTALL_PREFIX=/usr/local/gromacs-2024.6 -DGMX_GPU=CUDA -DCUDA_TOOLKIT_ROOT_DIR=/usr/local/cuda12.4

然后配置环境：

# chmod +x /usr/local/gromacs2024.6/bin/GMXRC
source /usr/local/gromacs2024.6/bin/GMXRC

能which到gmx说明成功。

GROMACS使用

主要针对有机小分子溶液相混合问题。

  1. 材料文件生成：确定体系内有哪些分子，Gview里准备好各物种的mol2文件，用Multiwfn的RESP.sh脚本做chg文件，拿到原子电荷。参考教程。
RESP.sh mol.mol2 0 1 gas
  1. 用Sobtop生成各分子拓扑文件。一个分子出三个文件。.top,.itp,.gro。参考官方教程 在生成文件时，先选7指定chg，就可以自动加入原子电荷。具体地，每一步选哪个选项，Sob老师已经写了，忘了去查。
指定chg：选 7 10

产生gro文件：选 2

产生itp文件：选 1 2 4

  1. 制作topol.top文件，记录整个混合系统的拓扑信息。几个关键点，一，要手动写[ default ]字段，具体参数要与使用的力场一致(打开力场文件复制即可)。二，将各itp文件的[ atom type]字段剪切到topol.top，位置在#include之前，去重，注意顺序要与itp文件中各原子出现顺序一致。三、重新规定Resname。方便后面分group。

  2. 制作盒子。用gmx制作盒子的过程。

gmx insert-molecules -ci mol.gro -nmol 10 -box 10 10 10 -o box.gro # 指定尺寸并加入一种gro
gmx insert-molecules -ci mol.gro -nmol 10 [-box 10 10 10] -f old_box.gro -o box.gro # 在一个盒子中加入另一种gro。-box可选，是用来增大尺寸。
#注：此处的gro不一定只含一个分子。

注意终端输出，实际加入数据可能小于指定数。如果报错，终端会提示。

  1. 显示溶剂的加入与加分子无差别。隐式溶剂的加入暂未考虑。离子的加入需要特殊说明。此处以KPF6为例。

（1）直接用RESP.sh脚本算不了盐体系，只能算出净电荷数为-1的PF6-阴离子。自然地，K+净电荷+1。但实际上有机盐做DFT计算出的RESP，阳离子通常只有+0.7~+0.95的电荷。所以最好是手动算，然后用Multiwfn计算RESP电荷。Multiwfn脚本从RESP.sh里扒一下就行了。 （2）DFT计算完，得到KPF6的log和fchk，gview获得mol2，Multiwfn算出chg，Sobtop算出gro，itp，top。这里的top没啥用，后面要做总的topol.top。 （3）从KPF6的itp里拆分出K+和PF6-的itp。PF6-.itp里删去和K+有关的所有参数。这样在拓扑上，K+与阴离子就没有成键项了。同时由于以盐的形式计算DFT，获得的原子电荷不会是+1。接近真实情况。 （4）根据研究的课题，有两种离子的插入方式。如果需要每一对阴阳离子保持紧密对结构，直接修改KPF6.gro里K的分子序号，原子序号，残基名即可。这样的gro维持了初始的盐的相对坐标。后续直接insert-molecules时，每个阴离子都在附近有阳离子。如果不关心这个问题，可以从KPF6.gro里拆出两个gro文件，再两次insert-molecules，可想而知，此时的阴阳离子都是均匀分布在盒子里的。 6. 能量最小化。

gmx grompp -f em.mdp -c box.gro -p topol.top -o em.tpr
gmx mdrun -deffnm em

mdp文件参考sob老师的模板，主要关注最小化方法(steep，cg...)，步长，步数，Fmax以及约束方式。 em的时候，注意读输出，Fmax并不一定总是能按照你的设想收敛，如果em做完势能依然很高，却盲目往下走，很可能后面体系要炸。 7. NVT与NPT NVT：

gmx grompp -f nvt.mdp -c em.gro -p topol.top -o nvt.tpr
gmx mdrun -v -deffnm nvt

NVT与NPT的区别就是是否开控压。 NPT：

gmx grompp -f npt.mdp -c nvt.gro -p topol.top -o npt.tpr
gmx mdrun -v -deffnm npt
  1. 几个问题：

" The X-size of the box (6.655719) times the triclinic skew factor (1.000000) is smaller than the number of DD cells (8) times the smallest allowed cell size(0.833498)。"

简单来讲，盒子太小、细分太高。

扩大盒子尺寸：

gmx editconf -f box.gro -o new_box.gro -box 7 7 7

或降低DD单元格数目：

gmx mdrun 里加上 -ntmpi 4

gmx dump -s md.tpr > 1.txt
# 查看盒子大小

gmx mdrun -v -deffnm -nb cpu -bonded cpu -pme cpu # 全程强制使用CPU计算，便于cuda报错时排查原因。
  1. VMD可视化：

先处理PBC设置，否则会出现奇怪条纹。

#如何将某个group平移到盒子中心？
gmx trjconv -s md.tpr -f md.trr -o md_center.trr -pbc cluster -center
#选择期望的group即可，但这样做完之后，依然不会改善条纹问题，所以再做一步-pbc mol是必要的。

gmx trjconv -s md.tpr -f md_center.trr -o md_fix.trr -pbc mol

# -pbc mol 保持每个分子整体不被切割，把主分子放到盒子中心。

加载gro文件后再加载trr轨迹文件即可。

vmd md.gro md_fix.trr 
#后续要手动删去第0帧，原因不言而喻。
  1. 输出一般数据，判断体系是否到达稳态，可据此调整步长和时间。

（1）体系能量：

gmx energy -f md.edr -o energy.xvg
# 2024.6版里，此命令后出现选择框，很多性质可一并输出。

（2）体系密度：

gmx density -s md.tpr -f md.trr

（3）RMSD 均方根偏差：

gmx rms -s md.tpr -f md.trr -o rmsd.xvg

（4）输出坐标：

gmx traj -s md.tpr -f md.trr -ox coords.xvg

上述命令均可加 -n index.ndx 使用定制化的索引文件。

（5）做索引文件：

gmx make_ndx -f md.tpr -o index.ndx
# 索引文件语法一例：
# r MOL & t os
# 意为选择resname==MOL 且 atomtype==os 的所有原子形成一个新的group
# 期望的groups全部创建好后，q保存退出

（6）RDF:

gmx rdf -f md.trr -s md.tpr -n index.ndx -o rdf.xvg -cn rdf_cn.xvg -bin 0.005 -b 1000 -e 2000 -rmax 1

#-f md.xtc：指定轨迹文件。 没有xtc用trr
#-s md.tpr：指定参数文件。
#-n index.ndx：指定索引文件，定义了感兴趣的原子组。
#-o rdf.xvg：输出RDF结果文件。
#-cn rdf_cn.xvg：输出配位数结果文件。
#-bin 0.01：设置RDF的bin宽度（nm）。
#-b 1000：设置计算开始时间（ps）。
#-e 2000：设置计算结束时间（ps）。
#-rmax 1：最大半径（nm）。
